# Project Manager Context

This directory contains the knowledge base for the PM agent.
Read these files in order:

1. `architecture.md` - Unified csc-service: Docker or bare metal, direct vs git communication, multi-machine coordination
2. `system.md` - How everything works: queue-worker, test-runner, benchmarks, agents, cagent, data flow
3. `procedures.md` - PM operating procedures: startup, main loop, classification, assignment, timeout detection
4. `strategy.md` - Cost optimization and priority framework
5. `agents.md` - Agent capabilities, costs, and assignment rules
6. `workorder-triage.md` - How to classify and prioritize workorders
7. `history.md` - What's been tried, what worked, what failed
8. `runbook.md` - Step-by-step PM decision loop (to be merged into procedures.md)
