## Summary
Preserve interrupted work, track finished vs unfinished tasks, and avoid rereading irrelevant history.

## Details
Davey is the primary user. He expects interrupted work to be stashed with enough context to resume later. He may assume a thread is finished unless unfinished state is recorded explicitly. Workorder document aesthetics are low priority as long as the message is clear.
