── irc/docs/KNOWN_ISSUES.txt ──
# 2026-03-08 | shared/server storage | os.replace() atomic rename is NOT atomic on Windows — the crash-safety guarantee (temp→fsync→rename) only holds on POSIX. On Windows a process kill mid-rename can leave a partial file. Affects channels.json, users.json, opers.json, bans.json, history.json. Fix: use a Windows-safe atomic write (e.g. write to temp, flush, close, then os.replace with retry logic or use a Win32 ReplaceFile call).
